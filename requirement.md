# 智能岗位聚合与匹配平台

## 需求规格说明书 v1.0

> 文档版本：v1.0 创建日期：2026-03-29 项目状态：需求阶段

------

## 一、项目概述

### 1.1 项目背景

在求职过程中，计算机科学及相关专业的求职者需要逐一访问国内各大互联网公司的招聘网站，手动浏览和筛选技术研发类岗位信息，流程繁琐且效率低下。国内公司的招聘网站没有通用的数据获取接口，页面结构各异，信息分散，难以横向对比。求职者迫切需要一个一站式平台，能够自动爬取并聚合多家国内公司的技术研发岗位信息，并提供智能分析与匹配功能，大幅提升求职效率。

### 1.2 项目目标

构建一个智能岗位聚合与匹配平台，核心目标包括：

- 自动爬取指定国内公司招聘网站的技术研发类岗位信息，统一结构化展示
- 基于用户简历和偏好，智能推荐最匹配的岗位
- 提供多岗位横向对比分析功能

### 1.3 目标用户

本平台主要面向计算机科学及相关专业的程序员和技术人才，尤其是正在寻找国内互联网公司技术研发类实习或全职岗位、需要同时关注多家公司招聘动态的求职者。

------

## 二、用户交互流程概述

用户的典型使用路径如下：

1. **浏览与收藏**：用户进入岗位总览页，先选择要查看的公司，然后逐一浏览该公司的岗位卡片。点击卡片查看岗位详情，对感兴趣的岗位进行收藏。用户可切换不同公司，重复此过程，积累多家公司的收藏岗位。
2. **智能匹配**（无明确意向时）：用户在浏览收藏了一些岗位后，进入智能匹配页。选择某家公司的收藏岗位作为分析范围，上传简历并填写个人偏好与要求，系统自动生成推荐报告，帮助用户从收藏中发现最适合的岗位。报告生成后，用户可在下方的对话框中继续向 LLM 追问，如深入了解某个岗位的细节、询问面试准备建议、调整偏好重新分析等。
3. **岗位对比**（有明确意向时）：用户已有几个意向岗位，进入岗位对比页。选择某家公司的收藏岗位作为分析范围，上传简历并填写个人偏好与要求，系统生成对比分析报告，帮助用户在意向岗位中做出最终选择。报告生成后，用户同样可通过对话框继续追问，如要求更细粒度的对比、询问某个岗位的技术栈学习路径等。

## 三、核心功能描述

### 3.1 岗位数据采集与聚合

#### 3.1.1 自动爬取采集

由于国内公司的招聘网站没有通用的数据获取接口，系统通过自动爬取的方式从各公司招聘页面采集岗位信息：

- 针对每家目标公司的招聘网站，编写对应的爬取逻辑（使用 Playwright 渲染页面 + Crawl4AI 提取内容）
- 用户可自定义添加目标公司列表，系统为新增公司适配爬取规则
- 定时自动采集（可配置采集频率，如每日/每12小时）
- 增量更新机制：仅采集新增或变动的岗位，避免重复采集
- 爬取到的原始内容通过 LLM 进行结构化解析，转换为统一的数据格式

#### 3.1.2 数据结构化处理

系统将采集到的原始岗位信息解析并结构化为统一格式，包含以下字段：

| 字段     | 说明                                 |
| -------- | ------------------------------------ |
| 岗位名称 | 职位标题                             |
| 岗位方向 | 所属技术方向（如算法、后端、前端等） |
| 所属公司 | 发布该岗位的公司名称                 |
| 工作地点 | 城市 / 远程 / 混合办公               |
| 岗位类型 | 实习 / 全职 / 兼职 / 合同工          |
| 核心职责 | 岗位的主要工作内容描述               |
| 技术要求 | 必备技能与加分技能（分级标注）       |
| 所属部门 | 该岗位所在的业务部门                 |
| 部门产品 | 该部门负责的主要产品或业务线         |
| 学历要求 | 最低学历要求                         |
| 经验要求 | 所需工作经验年限                     |
| 发布日期 | 岗位上线时间                         |
| 原始链接 | 跳转至公司官方招聘页面的链接         |

#### 3.1.3 采集岗位范围

系统仅采集技术研发类岗位，覆盖以下岗位方向：

| 岗位方向     | 典型职位示例                           |
| ------------ | -------------------------------------- |
| 算法         | 算法工程师、推荐算法、搜索算法         |
| 后端         | 后端开发工程师、Java 开发、Go 开发     |
| 客户端       | Android 开发、iOS 开发、桌面端开发     |
| 前端         | 前端开发工程师、Web 开发、全栈开发     |
| 测试         | 测试开发工程师、QA 工程师、自动化测试  |
| 大数据       | 数据工程师、数据仓库开发、Spark 开发   |
| 安全         | 安全工程师、渗透测试、安全研发         |
| 硬件         | 嵌入式开发、固件工程师、硬件设计       |
| 机器学习     | 机器学习工程师、深度学习研究员、MLOps  |
| 基础架构     | 基础架构工程师、平台开发、分布式系统   |
| 多媒体       | 音视频开发、流媒体工程师、图形引擎开发 |
| 计算机视觉   | CV 工程师、图像算法、3D 视觉           |
| 运维         | SRE、DevOps 工程师、云平台运维         |
| 数据挖掘     | 数据挖掘工程师、数据分析、用户增长     |
| 自然语言处理 | NLP 工程师、对话系统、文本挖掘         |

系统在采集时自动过滤非技术岗位（如产品、设计、运营、市场等），确保平台内容与目标用户高度相关。

------

### 3.2 岗位总览与浏览

#### 3.2.1 公司选择

岗位总览页顶部提供公司选择器，用户首先选择要查看的公司，页面随即展示该公司下的所有技术研发岗位。用户可随时切换公司。

#### 3.2.2 岗位筛选

在选定公司后，用户可通过以下维度进一步筛选岗位：

- 按岗位方向筛选（算法 / 后端 / 前端 / 机器学习 等，支持多选）
- 按工作地点筛选（城市 / 远程）
- 按岗位类型筛选（实习 / 全职 / 兼职）
- 按发布时间筛选（最近24小时 / 一周 / 一月）

#### 3.2.3 岗位卡片

以卡片视图展示岗位列表，每张卡片包含：岗位名称、岗位方向标签、工作地点、核心技术栈标签、发布时间、收藏按钮。

#### 3.2.4 全局搜索

- 支持关键词模糊搜索
- 搜索范围覆盖岗位名称、职责描述、技术要求等全部字段
- 支持自动补全和搜索建议

#### 3.2.5 岗位详情

点击岗位卡片后，右侧滑出详情面板，展示以下内容：

- 职位概述：一句话总结岗位核心内容
- 核心职责：该岗位的主要工作内容
- 技术要求：必备技能（红色标注）与加分技能（蓝色标注）
- 团队与产品：所属部门介绍及负责的主要产品
- 操作按钮：收藏 / 取消收藏、跳转原始链接

#### 3.2.6 岗位收藏

- 用户可在卡片列表或详情面板中收藏感兴趣的岗位
- 收藏按公司自动归类（如字节跳动收藏 3 个、美团收藏 5 个）
- 收藏数据在智能匹配和岗位对比模块中使用

------

### 3.3 智能匹配与推荐

智能匹配适用于用户尚无明确岗位意向的场景。系统从用户在某家公司收藏的所有岗位中，分析并推荐最适合的岗位。

#### 3.3.1 分析输入

用户在智能匹配页完成以下操作后发起分析：

1. **选择公司**：从下拉菜单中选择要分析哪家公司的收藏岗位
2. **上传简历**：支持 PDF / DOCX 格式，系统自动解析提取技能、经验、教育背景等信息
3. **填写偏好与要求**：用户输入自己的岗位兴趣方向、技术栈偏好、期望地点、岗位类型等个人偏好
4. **发起分析**：点击"生成推荐报告"按钮，系统开始处理

#### 3.3.2 报告生成

系统利用 LLM（大语言模型）对用户简历、偏好与所选公司的收藏岗位进行语义级别的分析，自动生成一份推荐报告。报告按推荐程度从高到低排列，每个岗位的分析内容包括：

- 推荐理由：为什么推荐该岗位，与用户技能、经验、偏好的契合点
- 岗位前景：该岗位的发展前景、团队实力、产品影响力等
- 技术栈分析：该岗位可能用到的技术栈，以及与用户现有技能的匹配情况
- 潜在不足：该岗位与用户预期可能存在的差距（如经验要求偏高、技术栈有未覆盖项等）

#### 3.3.3 报告展示

- 报告在页面内以文档形式呈现，按推荐优先级从高到低依次展示
- 用户可点击报告中的岗位名称跳转至岗位详情面板

#### 3.3.4 追问对话

报告生成后，页面下方显示 LLM 对话输入框，用户可以像使用普通 AI 聊天工具一样继续向系统提问。对话上下文自动携带用户简历、偏好以及已生成的报告内容，典型的追问场景包括：

- 深入了解某个推荐岗位的更多细节（如"推荐 #1 的团队规模大概多大？"）
- 针对某个岗位询问面试准备建议（如"如果投这个前端岗位，我应该重点准备什么？"）
- 调整偏好条件后要求重新分析（如"如果我不限制地点，排序会有变化吗？"）
- 询问技能提升路径（如"我该怎么快速补上 GraphQL？"）
- 其他任何与岗位选择相关的自由提问

------

### 3.4 岗位对比分析

岗位对比适用于用户已有若干意向岗位的场景。系统针对用户选定的岗位进行横向对比，帮助用户在已有意向中做出最优选择。

#### 3.4.1 分析输入

用户在岗位对比页完成以下操作后发起分析：

1. **选择公司**：从下拉菜单中选择要对比哪家公司的收藏岗位
2. **上传简历**：支持 PDF / DOCX 格式，系统自动解析
3. **填写偏好与要求**：用户输入个人偏好与岗位要求
4. **发起分析**：点击"生成对比报告"按钮，系统开始处理

#### 3.4.2 报告生成

系统利用 LLM 结合用户简历和偏好，对所选公司的收藏岗位进行对比分析，生成一份对比报告。报告按推荐程度从高到低排列，每个岗位的分析内容包括：

- 推荐理由：为什么该岗位排在当前位置，相比其他岗位的优势在哪里
- 岗位前景：该岗位的发展前景与成长空间
- 技术栈分析：该岗位涉及的技术栈，与其他岗位的差异
- 优劣势对比：该岗位相对于其他收藏岗位的优势与不足
- 综合建议：在报告末尾给出最终推荐意见

#### 3.4.3 报告展示

- 报告在页面内以文档形式呈现，按推荐优先级从高到低依次展示
- 用户可点击报告中的岗位名称跳转至岗位详情面板

#### 3.4.4 追问对话

报告生成后，页面下方显示 LLM 对话输入框，用户可继续追问。对话上下文自动携带用户简历、偏好以及已生成的对比报告内容，典型的追问场景包括：

- 要求对某两个岗位进行更细粒度的对比（如"全栈实习和前端岗位在日常工作节奏上有什么区别？"）
- 询问某个岗位的技术栈学习路径（如"如果选全栈岗位，我该怎么补 Java？"）
- 进一步了解岗位的团队或产品信息（如"美团的全栈团队主要负责哪些产品？"）
- 请求调整分析角度后重新给出建议（如"如果我更看重远程办公，推荐顺序会变吗？"）
- 其他任何与岗位对比和选择相关的自由提问
